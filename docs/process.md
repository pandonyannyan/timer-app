# Timer App — Progress

## Проект

Многопользовательское веб-приложение для управления таймерами внутри команды.

Основная идея:

- команда работает с общим списком таймеров;
- таймеры можно запускать, останавливать, перезапускать и завершать локально;
- часть глобальных действий доступна только пользователям с определённой ролью;
- каждый пользователь может закреплять важные для себя таймеры и менять порядок закреплённых;
- в будущем данные будут храниться в БД и синхронизироваться между пользователями.

Стек:

- Frontend: Vue 3 + Vite + TypeScript + Pinia
- Backend: планируется Python + FastAPI + Supabase
- Хостинг: GitHub Pages + бесплатные сервисы

Подробная архитектура, модели данных, роли, Pinia store и будущая backend/Supabase-структура описаны в [architecture.md](./architecture.md).

## Основные компоненты

### TimersPage.vue

Главная страница таймеров.

Отвечает за:

- подключение TimersToolbar;
- подключение TimersTable;
- получение таймеров из Pinia;
- поиск;
- фильтр по статусу;
- расчёт view-статуса;
- расчёт оставшегося времени;
- разделение таймеров на закреплённые и незакреплённые;
- автоматический порядок незакреплённых таймеров;
- режим перестановки закреплённых таймеров;
- draft-порядок закреплённых таймеров при drag-and-drop.

Важная логика:

- закреплённые таймеры всегда отображаются сверху;
- закреплённые таймеры отображаются в пользовательском порядке;
- порядок закреплённых таймеров хранится в pinnedTimerIds;
- незакреплённые таймеры отображаются по view-статусу:
  - signal;
  - active;
  - stopped;
  - completed;
- внутри одного view-статуса незакреплённые таймеры сортируются по оставшемуся времени;
- если время одинаковое, используется сортировка по названию как fallback;
- в режиме перестановки отображаются только закреплённые таймеры;
- в режиме перестановки поиск и фильтр сбрасываются и блокируются.

### TimersToolbar.vue

Панель над таблицей.

Содержит:

- фильтр “Все / Активные”;
- поиск;
- общий переключатель звука;
- кнопку “Изменить порядок” / “Сохранить порядок”;
- кнопку “Добавить таймер”.

Реализовано:

- кнопка добавления открывает AddTimerModal;
- кнопка изменения порядка работает только для закреплённых таймеров пользователя;
- кнопка изменения порядка disabled, если у пользователя меньше двух закреплённых таймеров;
- в reorder mode поиск и фильтры disabled;
- кнопка добавления disabled в reorder mode;
- общий переключатель звука включает/выключает звук для всех таймеров;
- переключатель использует статичные иконки выключенного и включённого звука по бокам от тумблера;
- права на создание таймеров учитываются через usePermissions;
- изменение порядка закреплённых не ограничивается ролью, потому что это персональная настройка пользователя.

### TimerRow.vue

Строка таймера.

Отвечает за:

- отображение кнопки закрепления или drag handle;
- отображение названия;
- изображение или placeholder;
- статус;
- описание;
- длительность;
- оставшееся время;
- дату, время, смещение и пользователя последнего запуска;
- кнопки действий.

Действия:

- закрепить / открепить таймер;
- перезапустить;
- остановить;
- включить/выключить звук для конкретного таймера;
- редактировать;
- удалить;
- завершить, если таймер в signal.

Реализовано:

- в обычном режиме первая колонка содержит pin-кнопку;
- в reorder mode первая колонка содержит drag handle, а pin-кнопка скрыта;
- открытие RestartTimerModal;
- открытие AddTimerModal в режиме редактирования;
- открытие ConfirmDeleteModal;
- disabled-состояния кнопок по ролям;
- disabled-состояния кнопок в reorder mode;
- информация о последнем запуске берётся из startedAt и lastRunBy;
- после перезапуска таймера в строке обновляются дата, время, смещение и пользователь последнего запуска;
- звук таймера проигрывается только если включён общий звук и включён звук конкретного таймера;
- при выключении общего звука проигрываемый сигнал останавливается;
- остановка звука при действиях;
- при удалении таймера очищается его локальное закрепление.

### RestartTimerModal.vue

Модалка повторного запуска таймера.

Режимы:

- запуск от текущего времени;
- запуск со сдвигом.

Реализовано:

- валидация целых минут;
- дефолтное смещение: 1 минута;
- смещение должно быть не меньше 1 минуты;
- смещение не может превышать базовую длительность таймера;
- кнопка запуска disabled при ошибке;
- аккуратная подсветка ошибки;
- Enter запускает таймер, если текущее состояние формы валидно;
- Esc закрывает модалку;
- фокус и disabled-состояния приведены к стилю приложения.

---

### AddTimerModal.vue

Универсальная модалка для создания и редактирования таймера.

Режимы:

- create
- edit

В режиме создания:

- поля пустые;
- после сохранения создаётся новый мок-таймер в Pinia.

В режиме редактирования:

- поля заполнены данными выбранного таймера;
- можно изменить:
  - название;
  - описание;
  - длительность;
  - изображение;
- можно удалить текущее изображение;
- после сохранения изменения применяются в Pinia.

Валидация:

- название обязательно;
- длительность обязательна;
- длительность должна быть целым числом минут;
- длительность должна быть от 1 минуты до 24 часов.

Изображение:

- если у таймера уже есть изображение, в режиме редактирования показывается кнопка удаления изображения;
- кнопка удаления использует тот же IconButton и deleteIcon, что и кнопка удаления таймера в TimerRow.vue;
- кнопка удаления расположена по правому краю строки выбора файла;
- удаление изображения не применяется сразу, а только помечается через removeImage;
- если пользователь нажмёт “Отмена”, изображение останется без изменений;
- если пользователь нажмёт “Сохранить”, updateTimer очистит imageUrl;
- если после удаления пользователь выберет новый файл, удаление отменяется и сохраняется новый файл.

Дополнительно:

- Esc закрывает модалку.

Важно:

- выбранное изображение сейчас превращается в URL.createObjectURL(file);
- это временное решение для фронта;
- после перезагрузки страницы такая картинка не сохранится;
- в будущем изображение нужно будет загружать в Supabase Storage или другой storage.

## Pinia store

Файл:

src/stores/timers.ts

Сейчас store содержит мок-данные и actions.

Реализованные actions:

- createTimer(payload)
- updateTimer(timerId, payload)
- deleteTimer(timerId)
- restartTimer(timerId, timeShiftSeconds)
- stopTimer(timerId)

Кратко:

- createTimer создаёт новый таймер локально;
- updateTimer редактирует существующий таймер локально;
- deleteTimer удаляет таймер и очищает локальный completed;
- restartTimer перезапускает таймер, обновляет данные последнего запуска и сбрасывает локальный completed;
- stopTimer останавливает таймер.

Подробности описаны в docs/architecture.md.

## Пользовательские закрепления

Реализовано через:

src/composables/usePinnedTimers.ts

Сейчас закрепления хранятся в localStorage.

Логика:

- закрепить таймер может любой пользователь;
- открепить таймер может любой пользователь;
- закреплённые таймеры всегда отображаются сверху;
- порядок закреплённых таймеров персональный для пользователя;
- порядок закреплённых хранится в массиве pinnedTimerIds;
- пользователь может менять порядок закреплённых таймеров через drag-and-drop;
- режим перестановки доступен только если закреплено больше одного таймера;
- при удалении таймера его закрепление очищается.

В будущем закрепления должны храниться в БД в таблице user_pinned_timers вместе с position.

## Роли и права

Текущие роли:

type UserRole = 'admin' | 'manager' | 'member'

Сейчас роль моковая в src/composables/usePermissions.ts.

Для проверки можно менять вручную:

const currentRole = ref<UserRole>('admin')
const currentRole = ref<UserRole>('manager')
const currentRole = ref<UserRole>('member')

Кратко:

- admin может выполнять все глобальные действия с таймерами и управлять пользователями;
- manager может создавать, редактировать, удалять, запускать и останавливать таймеры;
- member может запускать и останавливать таймеры;
- звук, поиск, фильтры, закрепления и порядок закреплённых таймеров считаются персональными действиями пользователя.

Важно:

- текущие права являются только UI-ограничением;
- настоящую безопасность нужно будет реализовать на backend/Supabase RLS.

## Порядок отображения таймеров

Глобальный ручной порядок таймеров удалён.

### Закреплённые таймеры

Закреплённые таймеры:

- всегда отображаются сверху;
- отображаются независимо от статуса;
- сортируются по пользовательскому порядку закрепления;
- могут быть переставлены пользователем через drag-and-drop.

### Незакреплённые таймеры

Незакреплённые таймеры отображаются автоматически по view-статусу:

- signal
- active
- stopped
- completed

Внутри одного view-статуса таймеры сортируются по оставшемуся времени.

Если оставшееся время одинаковое, используется сортировка по названию как fallback.

## Поиск и фильтры

Поиск ищет по:

- title;
- description.

Фильтры:

- all;
- active.

Фильтр active показывает:

- active;
- signal.

Поиск и фильтры применяются и к закреплённым, и к незакреплённым таймерам.

Если закреплённый таймер не подходит под поиск или фильтр, он не отображается в текущем результате.

## Что проверять после изменений

После любых крупных изменений запускать:

npm run build

Ручная проверка:

1. Таймеры отображаются.
2. Countdown работает.
3. Signal появляется при истечении таймера после открытия страницы.
4. Completed появляется для таймера, который истёк до открытия страницы.
5. Кнопка “Завершить” работает локально.
6. Перезапуск работает.
7. После перезапуска обновляются дата, время, смещение и пользователь последнего запуска.
8. Перезапуск со сдвигом валидируется.
9. Остановка работает.
10. Индивидуальное отключение звука у таймера работает.
11. Общее отключение звука для всех таймеров работает.
12. Если общий звук выключен, сигнал таймеров не проигрывается.
13. Если общий звук выключить во время сигнала, звук останавливается.
14. После перезагрузки страницы настройки звука сохраняются.
15. Добавление таймера работает.
16. Редактирование таймера работает.
17. Валидация длительности таймера при добавлении и редактировании работает: 0 и 1441 минут не сохраняются, 1 и 1440 минут сохраняются.
18. Удаление изображения таймера в режиме редактирования работает.
19. Удаление таймера работает.
20. Закрепление таймера работает.
21. Открепление таймера работает.
22. Закреплённые таймеры отображаются сверху.
23. Закреплённые таймеры сохраняются после перезагрузки страницы.
