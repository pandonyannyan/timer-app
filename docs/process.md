# Timer App — Progress

## Проект

Timer App — многопользовательское веб-приложение для управления таймерами внутри команды.

Основная идея:

- команда работает с общим списком таймеров;
- таймеры можно запускать, останавливать, перезапускать и завершать локально;
- глобальные действия ограничиваются ролью пользователя;
- каждый пользователь может закреплять важные для себя таймеры и менять порядок закреплённых;
- в будущем данные будут храниться в БД и синхронизироваться между пользователями.

Стек:

- Frontend: Vue 3 + Vite + TypeScript + Pinia
- Backend: планируется Python + FastAPI + Supabase
- Хостинг: GitHub Pages + бесплатные сервисы

Подробная архитектура, модели данных, роли, Pinia store и будущая backend/Supabase-структура описаны в `docs/architecture.md`.

## Текущий статус

На текущем этапе реализована основная frontend-логика приложения.

Backend пока не подключён. Таймеры работают через mock-реализацию в `src/services/timerService.ts`, а Pinia store использует этот сервис как промежуточный слой перед будущим API.

Пользовательские настройки пока частично сохраняются в `localStorage` через отдельные services/utilities.

Первый этап подготовки к MVP завершён: frontend service layer вынесен для таймеров, закреплений и настроек звука.

## Основные компоненты

### TimersPage.vue

Главная страница таймеров.

Отвечает за:

- подключение `TimersToolbar`;
- подключение `TimersTable`;
- получение таймеров из Pinia;
- поиск;
- фильтр по статусу;
- расчёт view-статуса;
- расчёт оставшегося времени;
- разделение таймеров на закреплённые и незакреплённые;
- автоматический порядок незакреплённых таймеров;
- режим перестановки закреплённых таймеров;
- draft-порядок закреплённых таймеров при drag-and-drop.

Важная логика:

- закреплённые таймеры всегда отображаются сверху;
- закреплённые таймеры отображаются в пользовательском порядке;
- незакреплённые таймеры отображаются по view-статусу: `signal`, `active`, `stopped`, `completed`;
- внутри одного view-статуса незакреплённые таймеры сортируются по оставшемуся времени;
- если время одинаковое, используется сортировка по названию;
- в режиме перестановки отображаются только закреплённые таймеры;
- в режиме перестановки поиск и фильтр сбрасываются и блокируются.

### TimersToolbar.vue

Панель над таблицей.

Содержит:

- фильтр “Все / Активные”;
- поиск;
- общий переключатель звука;
- кнопку “Изменить порядок” / “Сохранить порядок”;
- кнопку “Добавить таймер”.

Особенности:

- кнопка изменения порядка активна только если у пользователя больше одного закреплённого таймера;
- в reorder mode поиск, фильтры и добавление таймера disabled;
- права на создание таймеров учитываются через `usePermissions`;
- изменение порядка закреплённых не ограничивается ролью, потому что это персональная настройка пользователя.

### TimersTable.vue

Таблица таймеров.

Отвечает за:

- отображение списка `TimerRow`;
- empty state;
- drag-and-drop режим перестановки закреплённых таймеров.

В обычном режиме таблица отображает переданный список. В reorder mode строки становятся draggable, а новый порядок передаётся наверх через emit.

### TimerRow.vue

Строка таймера.

Отвечает за:

- отображение закрепления или drag handle;
- отображение названия, изображения, статуса, описания, длительности и оставшегося времени;
- отображение информации о последнем запуске;
- действия с таймером.

Действия:

- закрепить / открепить таймер;
- перезапустить;
- остановить;
- включить/выключить звук конкретного таймера;
- редактировать;
- удалить;
- завершить, если таймер в `signal`.

Звук проигрывается только если включён общий звук и включён звук конкретного таймера.

### AddTimerModal.vue

Универсальная модалка создания и редактирования таймера.

Валидация:

- название обязательно;
- длительность обязательна;
- длительность должна быть целым числом минут;
- длительность должна быть от 1 минуты до 24 часов.

Изображение:

- в режиме редактирования можно удалить текущее изображение;
- удаление применяется только после нажатия “Сохранить”;
- если после удаления выбрать новый файл, сохраняется новый файл.

### RestartTimerModal.vue

Модалка повторного запуска таймера.

Режимы:

- запуск от текущего времени;
- запуск со сдвигом.

Валидация сдвига:

- значение должно быть целым числом минут;
- значение должно быть не меньше 1 минуты;
- значение не должно превышать базовую длительность таймера.

Дополнительно:

- Enter запускает таймер, если форма валидна;
- Esc закрывает модалку.

### ConfirmDeleteModal.vue

Модалка подтверждения удаления.

Сценарий:

- пользователь нажимает “Удалить”;
- открывается подтверждение;
- “Отмена” закрывает модалку;
- “Удалить” удаляет таймер;
- локальный `completed` для таймера очищается;
- локальное закрепление таймера очищается.

## Pinia store

Файл:

`src/stores/timers.ts`

Сейчас store содержит состояние таймеров и actions.

Работа с mock-таймерами вынесена в:

`src/services/timerService.ts`

Это промежуточный слой перед подключением API: компоненты продолжают работать через Pinia, но основная mock-логика больше не находится внутри store.

Actions:

- `loadTimers()`
- `createTimer(payload)`
- `updateTimer(timerId, payload)`
- `deleteTimer(timerId)`
- `restartTimer(timerId, timeShiftSeconds)`
- `stopTimer(timerId)`

Кратко:

- `createTimer` создаёт новый таймер через сервис и добавляет его в state;
- `updateTimer` обновляет существующий таймер;
- `deleteTimer` удаляет таймер и очищает локальный `completed`;
- `restartTimer` перезапускает таймер, обновляет данные последнего запуска и очищает локальный `completed`;
- `stopTimer` останавливает таймер.

## Пользовательские закрепления

Файлы:

- `src/composables/usePinnedTimers.ts`
- `src/services/pinnedTimerService.ts`

Сейчас закрепления хранятся в `localStorage`, но работа с хранилищем вынесена в `pinnedTimerService`.

`usePinnedTimers` отвечает за реактивное состояние и методы для UI.

Логика:

- закрепить таймер может любой пользователь;
- открепить таймер может любой пользователь;
- закреплённые таймеры всегда отображаются сверху;
- порядок закреплённых таймеров персональный для пользователя;
- порядок закреплённых хранится в массиве `pinnedTimerIds`;
- пользователь может менять порядок закреплённых через drag-and-drop;
- режим перестановки доступен только если закреплено больше одного таймера;
- при удалении таймера его закрепление очищается.

В будущем закрепления должны храниться в БД в таблице `user_pinned_timers` вместе с `position`.

## Настройки звука

Файл:

`src/services/soundSettingsService.ts`

Сейчас настройки звука сохраняются в `localStorage` через сервис.

Есть два уровня настроек:

- общий звук для всех таймеров;
- звук конкретного таймера.

Итоговое правило:

`globalSoundEnabled && timerSoundEnabled`

Звук проигрывается только если общий звук включён и включён звук конкретного таймера.

Если общий звук выключить во время проигрывания сигнала, текущий звук останавливается.

В будущем сервис можно заменить на работу с пользовательскими настройками в БД.

## Локально завершённые таймеры

Файл:

`src/utils/completedTimers.ts`

Локальное завершение используется для сценария “Завершить”.

Если пользователь нажал “Завершить” у таймера в состоянии `signal`, локально сохраняется ключ конкретного запуска таймера:

`timerId + startedAt`

Это нужно, чтобы после перезапуска старый локальный `completed` не применялся к новому запуску того же таймера.

При перезапуске или удалении таймера локальный `completed` очищается.

## Роли и права

Текущие роли:

`admin`, `manager`, `member`

Сейчас роль моковая в `src/composables/usePermissions.ts`.

Кратко:

- `admin` может выполнять все глобальные действия с таймерами и управлять пользователями;
- `manager` может создавать, редактировать, удалять, запускать и останавливать таймеры;
- `member` может запускать и останавливать таймеры;
- звук, поиск, фильтры, закрепления и порядок закреплённых таймеров считаются персональными действиями пользователя.

Важно:

- текущие права являются только UI-ограничением;
- настоящую безопасность нужно будет реализовать на backend/Supabase RLS.

## Порядок отображения таймеров

Глобальный ручной порядок таймеров удалён.

### Закреплённые таймеры

Закреплённые таймеры:

- всегда отображаются сверху;
- отображаются независимо от статуса;
- сортируются по пользовательскому порядку закрепления;
- могут быть переставлены пользователем через drag-and-drop.

### Незакреплённые таймеры

Незакреплённые таймеры отображаются автоматически по view-статусу:

1. `signal`
2. `active`
3. `stopped`
4. `completed`

Внутри одного view-статуса таймеры сортируются по оставшемуся времени.

Если оставшееся время одинаковое, используется сортировка по названию.

## Поиск и фильтры

Поиск ищет по:

- title;
- description.

Фильтры:

- all;
- active.

Фильтр `active` показывает:

- `active`;
- `signal`.

Поиск и фильтры применяются и к закреплённым, и к незакреплённым таймерам.

Если закреплённый таймер не подходит под поиск или фильтр, он не отображается в текущем результате.

## Что проверять после изменений

После любых крупных изменений запускать:

`npm run build`

Ручная проверка:

1. Таймеры отображаются.
2. Countdown работает.
3. Signal появляется при истечении таймера после открытия страницы.
4. Completed появляется для таймера, который истёк до открытия страницы.
5. Кнопка “Завершить” работает локально.
6. Перезапуск работает.
7. После перезапуска обновляются дата, время, смещение и пользователь последнего запуска.
8. Перезапуск со сдвигом валидируется.
9. Остановка работает.
10. Индивидуальное отключение звука у таймера работает.
11. Общее отключение звука для всех таймеров работает.
12. Если общий звук выключен, сигнал таймеров не проигрывается.
13. Если общий звук выключить во время сигнала, звук останавливается.
14. После перезагрузки страницы настройки звука сохраняются.
