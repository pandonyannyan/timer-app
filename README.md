# Timer App

Timer App — веб-приложение для командной работы с таймерами.

Пользователи видят общий список таймеров, отслеживают обратный отсчёт, получают звуковые уведомления по истечении времени и выполняют действия с таймерами в зависимости от своей роли.

Проект рассчитан на небольшую команду и строится с учётом бесплатной инфраструктуры.

---

## Текущий статус

На текущем этапе реализована основная frontend-логика приложения.

Backend пока не подключён. Данные хранятся в Pinia как мок-данные, поэтому после перезагрузки страницы добавленные, удалённые, отредактированные таймеры и изменённый порядок возвращаются к исходному мок-состоянию.

Реализовано:

- таблица таймеров;
- realtime countdown на фронте;
- статусы `active`, `stopped`, `signal`, `completed`;
- добавление, редактирование и удаление таймеров;
- удаление изображения таймера в режиме редактирования;
- перезапуск таймера, в том числе со сдвигом;
- валидация сдвига от `1` минуты до длительности таймера;
- остановка и локальное завершение таймера;
- звуковой сигнал;
- индивидуальное отключение звука у конкретного таймера;
- общее отключение звука для всех таймеров;
- роли `admin`, `manager`, `member` на уровне UI;
- drag-and-drop перестановка таймеров;
- сохранение порядка через `sortOrder` в Pinia.

## Роли

| Роль | Доступ |
|---|---|
| `admin` | Все действия с таймерами + управление пользователями |
| `manager` | Все действия с таймерами, кроме изменения порядка |
| `member` | Запуск и остановка таймеров |

Сейчас роли реализованы только на frontend-уровне через `usePermissions.ts`.

В будущем ограничения должны быть продублированы на backend/Supabase RLS.

---

## Ограничения текущего этапа

- нет backend;
- нет авторизации;
- нет настоящих пользователей;
- нет синхронизации между пользователями;
- роли моковые;
- действия с таймерами меняют только Pinia;
- настройки звука пока сохраняются в `localStorage`, позже должны храниться в БД как пользовательские настройки;
- изображения отображаются через `URL.createObjectURL(file)` и не сохраняются после перезагрузки;
- изменения не сохраняются после перезагрузки страницы.
