# Timer App

Timer App — веб-приложение для командной работы с таймерами.

Пользователи видят общий список таймеров, отслеживают обратный отсчёт, получают звуковые уведомления по истечении времени и выполняют действия с таймерами в зависимости от своей роли.

Приложение рассчитано на небольшую команду и строится с учётом бесплатной инфраструктуры.

---

## Текущий статус

На текущем этапе реализована основная frontend-логика приложения.

Backend пока не подключён. Данные таймеров хранятся в Pinia как мок-данные, поэтому после перезагрузки страницы добавленные, удалённые и отредактированные таймеры возвращаются к исходному мок-состоянию.

Пользовательские настройки сейчас частично сохраняются в `localStorage`.

Реализовано:

- таблица таймеров;
- realtime countdown на фронте;
- статусы `active`, `stopped`, `signal`, `completed`;
- автоматический порядок незакреплённых таймеров: `signal`, `active`, `stopped`, `completed`;
- добавление, редактирование и удаление таймеров;
- удаление изображения таймера в режиме редактирования;
- перезапуск таймера, в том числе со сдвигом;
- управление модальными окнами с клавиатуры: `Esc` для закрытия, `Enter` для запуска в модалке перезапуска;
- обновление информации о последнем запуске таймера при перезапуске;
- валидация сдвига от `1` минуты до длительности таймера;
- остановка и локальное завершение таймера;
- звуковой сигнал;
- индивидуальное отключение звука у конкретного таймера;
- общее отключение звука для всех таймеров;
- роли `admin`, `manager`, `member` на уровне UI;
- персональное закрепление таймеров пользователем;
- drag-and-drop перестановка закреплённых таймеров;
- сохранение закреплений и их порядка в `localStorage`.

## Роли

| Роль | Доступ |
|---|---|
| `admin` | Все действия с таймерами + управление пользователями |
| `manager` | Все действия с таймерами |
| `member` | Запуск, остановка и персональные действия с таймерами |

Сейчас роли реализованы только на frontend-уровне через `usePermissions.ts`.

Персональные действия пользователя не ограничиваются ролью:

- поиск;
- фильтр;
- включение и выключение звука;
- закрепление таймеров;
- изменение порядка закреплённых таймеров.

В будущем ограничения для глобальных действий должны быть продублированы на backend/Supabase RLS.

---

## Ограничения текущего этапа

- нет backend;
- нет авторизации;
- нет настоящих пользователей;
- нет синхронизации между пользователями;
- роли моковые;
- действия с таймерами меняют только Pinia;
- настройки звука пока сохраняются в `localStorage`, позже должны храниться в БД как пользовательские настройки;
- закрепления таймеров и порядок закреплённых таймеров пока сохраняются в `localStorage`, позже должны храниться в БД как пользовательские настройки;
- изображения отображаются через `URL.createObjectURL(file)` и не сохраняются после перезагрузки;
- изменения таймеров не сохраняются после перезагрузки страницы.
